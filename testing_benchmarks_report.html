📐 Testing Methodology

Accuracy: We planted 18 known bugs across 3 files (SQL injection, XSS, path traversal, memory leaks, code smells, design issues). Each bug has a documented line number, category, and severity. The AI's findings are matched against ground truth with ±5 line tolerance.
Precision: TP / (TP + FP) — measures signal-to-noise ratio. High precision = fewer false alarms.
Recall: TP / (TP + FN) — measures bug coverage. High recall = fewer missed vulnerabilities.
F1 Score: Harmonic mean of Precision and Recall — the single best accuracy metric.
Efficiency: Wall-clock time per file scan (includes API latency + prompt processing).
Reliability: Each file is scanned 3 times. Jaccard similarity between run results measures consistency. 100% = identical findings every run.
Fix Quality: Percentage of detected bugs that include a compilable fixSnippet (drop-in replacement code).
Impact: Critical/High bugs detected vs missed — weighted by severity to measure real-world risk reduction.
